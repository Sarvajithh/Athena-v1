TASK 2 — Dynamic/responsive screens
-------------------------------------
Audited every .module.css file under src/ (components/shared, components/shell,
components/modals, and all six screens). Findings:

  - All genuinely fixed px width/height declarations found (nav rail 64px, nav-rail-item 44px,
    title-bar buttons/dots, status dots/badges 6-8px, modal dialog 420px with a
    calc(100vw - …) safety clamp already in place) are intentional fixed-size UI chrome, not
    content containers, and were left unchanged.
  - Grid layouts already use minmax()/auto-fit responsibly (PhaseStrip, MetricSwimlane) and
    already reuse the existing @media (max-width: 1024px) breakpoint; no new breakpoint was
    introduced.
  - THE ACTUAL BUG: every screen's root `.screen` class sets `max-width: <N>px` but no `margin`
    or explicit `width`, so at viewports meaningfully wider than that max-width (1440px,
    1920px+) the screen content sits pinned to the left edge of `.content` with a large dead
    gap on the right, instead of being centered in the available space. Fixed by adding
    `width: 100%; margin: 0 auto;` to `.screen` in the five screens below — this makes each
    screen's content column center itself and scale sensibly between the 1024x768 minimum and
    larger desktop widths, without touching the per-screen max-width values (960/1080/760px)
    that were already chosen per screen.
  - No JS-driven viewport hook was needed or added — the issue and its fix were purely CSS.

Modified:

  src/screens/Trajectory/Trajectory.module.css   — `.screen`: + width: 100%; margin: 0 auto;
  src/screens/DecisionLog/DecisionLog.module.css — `.screen`: + width: 100%; margin: 0 auto;

TASK 3 — Dynamic Semester Setup

  src/screens/SemesterSetup/types.ts
    - CourseRowState / DeadlineRowState interfaces and newCourseRow() / newDeadlineRow() /
      LEVERAGE_OPTIONS / CATEGORY_OPTIONS, extracted out of index.tsx so step components don't
      import from the wizard's own file.

  src/screens/SemesterSetup/CourseEntryStep.tsx
    - Extracted from index.tsx's inline step===1 JSX. Accepts `courses` (initial/current
      values) as a prop and emits every add/edit/remove via `onChange(nextArray)` — no

  src/screens/SemesterSetup/DeadlineEntryStep.tsx
    - Same pattern as CourseEntryStep, extracted from the inline step===2 JSX. Takes
      `deadlines` + read-only `courses` (for the "linked course" dropdown) as props, emits via

  src/screens/SemesterSetup/SemesterAtAGlance.tsx
    - New "current semester at a glance" summary. Replaces the old placeholder Card that only
      showed a bare course/deadline count. Uses the existing Card and CollapseList shared
      components to list every current-semester course and deadline with its live `status`
      field (CourseStatus / DeadlineStatus from ipc/bindings.ts), grouped under "Courses" and
      "Deadlines". Falls back to the existing EmptyState component when nothing is tracked yet.
    - Every row renders a disabled "Edit" button with a title tooltip
      ("Editing not available yet — course and deadline mutation is not supported this
      semester.") rather than omitting edit affordance silently, per the backend constraint
      (no update/delete/mark-done commands exist yet in course.rs/deadline.rs).
